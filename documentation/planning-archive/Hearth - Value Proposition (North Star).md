# Hearth — Value Proposition (North Star)

*Keep this front and centre. Every feature, every decision, every "should we build X?" gets checked against this.*

---

## The problem in one sentence

**Our community's trusted knowledge — who the good practitioners are, and what's happening — keeps evaporating, because it only lives in WhatsApp (which forgets) and a clunky calendar (which nobody reads on a phone).**

That knowledge has two defining properties:

- **Ephemeral** — a great recommendation scrolls away in a day; an event vanishes once the message is buried.
- **Trust-bound** — what made the recommendation valuable was that *someone in our circle vouched for them*. Strip out the trust and it's just a name.

So the problem is **not** "we lack a directory." It's: *there is no lasting, searchable, curated home for the knowledge this community already holds.*

---

## Why it isn't solved today

The two halves have different answers — and being honest about this keeps us from rebuilding things that already exist.

**Events — largely a solved commodity.** Luma already does community calendars, RSVPs, reminders, and discovery, for free. WhatsApp has a native Events feature too. So we should **lean on / link to existing tools for events, not build our own.** (Telling detail: WhatsApp Events can't show new members events created before they joined — proof that WhatsApp is structurally built to *forget*. Great at the live moment, bad at memory.)

**Directory — solved generically, but never for *us*.** Plenty of holistic-practitioner directories exist (Heallist, BodyMindSpirit, SoulSearch, and many more). But every one is **global, commercial, and modality-organized** — a stranger's listing you have no reason to trust. None of them is *our* community. None carries the only signal that matters here: **"this person is one of us, recommended by people you know."**

---

## The unique value proposition

> **Hearth is the lasting, searchable home for the practitioners and events that *our* community vouches for.**

The thing no existing product delivers is the **intersection**:

- *Persistent + searchable* (which WhatsApp can never be), **for**
- *a specific, trusted, curated set of people* (which generic directories never are).

Luma owns events. Heallist owns global directories. WhatsApp owns the live, trusted moment. **Nobody owns "the lasting, searchable directory of practitioners our community trusts."** That gap is Hearth's entire reason to exist.

**The moat is not software.** It's the trusted membership and the curation. A better-built directory can't replicate "vetted by our community" — which is exactly why the abundance of directory tools out there doesn't close the gap.

---

## What this means for the product

1. **The directory is the "why."** It's the defensible, unique core. Protect it. Invest there.
2. **Events are a commodity.** Lean on Luma / the existing calendar. Don't pour effort into rebuilding event infrastructure.
3. **WhatsApp and Hearth do different jobs.** WhatsApp = the live moment. Hearth = the durable memory the moment flows into. Don't make Hearth compete for the moment; make it the place the moment gets *saved*.

---

## The honest caveat (so we never forget it)

The directory problem is **real but not acute.** WhatsApp recommendations work "well enough," and people only need a practitioner occasionally — so the pain is mild and intermittent. The problem is genuine; it's just not on fire.

That's why **adoption is the whole game.** The bet isn't "is this worth solving" — it's *"can we make the trusted, persistent home so low-friction that people use it instead of letting the knowledge evaporate."*

**Every decision should be measured against that bet: does this make Hearth lower-friction and more trusted, or just bigger?**

---

## The one-line test

Before building anything, ask:

> *Does this strengthen the lasting, trusted, searchable home for our community's practitioners — or are we rebuilding something Luma/WhatsApp already does?*
